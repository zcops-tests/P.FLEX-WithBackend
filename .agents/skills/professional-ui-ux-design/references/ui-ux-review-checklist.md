# UI UX Review Checklist

Use this checklist to review a screen, PR, or redesign proposal.

## 1. First impression

- Does the screen feel intentional within the first few seconds?
- Is the main action immediately obvious?
- Does the visual style fit the product and audience?

## 2. Hierarchy and scanability

- Can the user identify the headline, supporting context, and primary action quickly?
- Is information grouped clearly?
- Does spacing create structure without needing excessive borders?
- Are there too many elements competing for attention?

## 3. Task flow

- Is the path to the main task short and understandable?
- Does the interface reduce unnecessary decisions?
- Is the next best action visible in empty, blocked, or success states?

## 4. Interaction quality

- Are hover, focus, disabled, loading, and pressed states defined?
- Is feedback immediate and understandable?
- Are destructive actions clearly distinguished and confirmed when appropriate?

## 5. Content clarity

- Are labels specific and outcome-oriented?
- Are helper texts and empty states useful?
- Is jargon removed unless it is appropriate for the audience?

## 6. Responsiveness and accessibility

- Does the layout still work on mobile widths?
- Are tap targets large enough?
- Is the screen keyboard navigable?
- Is important meaning preserved without relying on color alone?
- Is contrast sufficient for key text and actions?

## 7. Trust and polish

- Does the UI feel stable, calm, and credible?
- Are visual accents used intentionally rather than everywhere?
- Do loading and error states feel integrated into the design?

## 8. Merge gate

Treat the design as incomplete if any of these are missing without justification:

- clear hierarchy
- a single obvious primary action
- designed loading, empty, and error states
- accessibility for focus, contrast, and keyboard use
- responsive behavior for common screen sizes
